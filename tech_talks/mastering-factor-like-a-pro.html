TL; DR

Factor is an integer vector with a different class name and categorical levels. Factors often confuse R users in ways that they look and sometimes behave like characters, however they are actually integers underneath. In this session, we will talk about how to utilize factors while programming with R.

What is a factor

This is a regular vector

friends_characters <- c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing", "Ross Geller")
class(friends_characters)
## [1] "character"
friends_characters
## [1] "Rachel Green"   "Monica Geller"  "Phoebe Buffay"  "Joey Tribbiani"
## [5] "Chandler Bing"  "Ross Geller"

This is a factor

friends_characters <- factor(c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing", "Ross Geller"))
class(friends_characters)
## [1] "factor"
friends_characters
## [1] Rachel Green   Monica Geller  Phoebe Buffay  Joey Tribbiani
## [5] Chandler Bing  Ross Geller   
## 6 Levels: Chandler Bing Joey Tribbiani Monica Geller ... Ross Geller

A factor is a vector that can contain only predefined values and is used to store categorical data

Factors are built on top of an integer vector with two attributes

  • a different class name
  • levels
friends_characters <- factor(c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing", "Ross Geller"))
str(attributes(friends_characters))
## List of 2
##  $ levels: chr [1:6] "Chandler Bing" "Joey Tribbiani" "Monica Geller" "Phoebe Buffay" ...
##  $ class : chr "factor"

To be beginner friendly, just think factor as a named integer vector

lvs <- c("Chandler Bing", "Joey Tribbiani", "Monica Geller", "Phoebe Buffay", "Rachel Green", "Ross Geller")
friends_characters <- setNames(1L:6L, lvs)
friends_characters
##  Chandler Bing Joey Tribbiani  Monica Geller  Phoebe Buffay   Rachel Green 
##              1              2              3              4              5 
##    Ross Geller 
##              6

To be technically correct, we can create a factor from scratch like this

lvs <- c("Chandler Bing", "Joey Tribbiani", "Monica Geller", "Phoebe Buffay", "Rachel Green", "Ross Geller")
friends_characters <- structure(
  1L:6L,
  levels = lvs,
  class = "factor"
)
str(attributes(friends_characters))
## List of 2
##  $ levels: chr [1:6] "Chandler Bing" "Joey Tribbiani" "Monica Geller" "Phoebe Buffay" ...
##  $ class : chr "factor"

factor() function has 2 useful parameters

The first one would be levels

friends_vec <- c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing", "Ross Geller")
friends_characters <- factor(friends_vec)
levels(friends_characters) # alphabetical order
## [1] "Chandler Bing"  "Joey Tribbiani" "Monica Geller"  "Phoebe Buffay" 
## [5] "Rachel Green"   "Ross Geller"

levels allows users customizing levels of encoding

friends_characters <- factor(friends_vec, levels = friends_vec)
levels(friends_characters) # customized order
## [1] "Rachel Green"   "Monica Geller"  "Phoebe Buffay"  "Joey Tribbiani"
## [5] "Chandler Bing"  "Ross Geller"

The second one would be ordered

friends_characters <- factor(c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing", "Ross Geller"))
friends_characters
## [1] Rachel Green   Monica Geller  Phoebe Buffay  Joey Tribbiani
## [5] Chandler Bing  Ross Geller   
## 6 Levels: Chandler Bing Joey Tribbiani Monica Geller ... Ross Geller
friends_characters[1] > friends_characters[2]
## Warning in Ops.factor(friends_characters[1], friends_characters[2]): '>'
## not meaningful for factors
## [1] NA

ordered makes factor ordinal

friends_characters <- factor(c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing", "Ross Geller"), ordered = TRUE)
friends_characters
## [1] Rachel Green   Monica Geller  Phoebe Buffay  Joey Tribbiani
## [5] Chandler Bing  Ross Geller   
## 6 Levels: Chandler Bing < Joey Tribbiani < ... < Ross Geller
friends_characters[1] > friends_characters[2] # 5 > 3
## [1] TRUE

In base R we tend to encounter factors very frequently

The biggest challenge for R beginners using factor

Factors look like characters, however they are actually integers in disguise.

We can use nchar() on character vectors nice and easy

friends_characters <- c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing", "Ross Geller")
nchar(friends_characters)
## [1] 12 13 13 14 13 11

However, if we apply nchar() on factor vectors

friends_characters <- factor(c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing", "Ross Geller"))
nchar(friends_characters)
## Error in nchar(friends_characters): 'nchar()' requires a character vector

We can also use c() on character vectors nice and easy

friends_characters <- c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing")
friends_characters <- c(friends_characters, "Ross Geller")
friends_characters
## [1] "Rachel Green"   "Monica Geller"  "Phoebe Buffay"  "Joey Tribbiani"
## [5] "Chandler Bing"  "Ross Geller"

However, if we apply c() on factor vectors

friends_characters <- factor(c("Rachel Green", "Monica Geller", "Phoebe Buffay", "Joey Tribbiani", "Chandler Bing"))
friends_characters <- c(friends_characters, "Ross Geller")
friends_characters
## [1] "5"           "3"           "4"           "2"           "1"          
## [6] "Ross Geller"

It seems that factor has tons of disadvantages

Why does stringsAsFactors not default to FALSE

  • R was primarily being used by statisticians in the old days
  • Because of modeling functions like lm() and glm()

R is being used a by a very wide variety of people doing all kinds of things nowadays

Most often, people complaining about factors are people who are doing things that are not the traditional statistical modeling things

Modeling functions need to treat expand categorical variables into individual dummy variables

df <- read.csv("https://kaggle-getting-started.s3-ap-northeast-1.amazonaws.com/titanic/train.csv")
fit <- glm(Survived ~ Sex, data = df, family = binomial(link = "logit"))
fit$coefficients
## (Intercept)     Sexmale 
##    1.056589   -2.513710

What if we do exactly the same in Python

import pandas as pd
from sklearn.linear_model import LogisticRegression
df = pd.read_csv("https://kaggle-getting-started.s3-ap-northeast-1.amazonaws.com/titanic/train.csv")
X = df["Sex"].values.reshape(-1, 1)
y = df["Survived"].values
lr = LogisticRegression()
lr.fit(X, y)
## ValueError: could not convert string to float: 'male'

In Python we manually create dummy variables

import pandas as pd
from sklearn.linear_model import LogisticRegression
df = pd.read_csv("https://kaggle-getting-started.s3-ap-northeast-1.amazonaws.com/titanic/train.csv")
X = pd.get_dummies(df["Sex"].values, drop_first=True)["male"].values.reshape(-1, 1)
y = df["Survived"].values
lr = LogisticRegression(solver='lbfgs', penalty='none')
lr.fit(X, y)
print(lr.intercept_)
print(lr.coef_)
## [1.05659211]
## [[-2.51371812]]

Here’s a question, how does factor relate to one-hot encoding?

Let’s assume we have to do one-hot encoding from scratch

get_dummies <- function(chr_vector) {
  # A mapping list
  lvs <- sort(unique(chr_vector))
  char_to_int <- list()
  for (i in 1:length(lvs)) {
    list_idx <- lvs[i]
    char_to_int[[list_idx]] <- i
  }
  # One-hot encode
  one_hot_encoded <- data.frame(matrix(0, nrow = length(chr_vector), ncol = length(lvs)))
  colnames(one_hot_encoded) <- lvs
  for (i in 1:length(chr_vector)) {
    list_idx <- chr_vector[i]
    col_idx <- char_to_int[[list_idx]]
    one_hot_encoded[i, col_idx] <- 1
  }
  return(one_hot_encoded)
}
df <- read.csv("https://kaggle-getting-started.s3-ap-northeast-1.amazonaws.com/titanic/train.csv")
sex_chr <- as.character(df$Sex)
ohe <- get_dummies(sex_chr)

We no longer need creating the mapping list ourselves if we have factor in hand

get_dummies <- function(factor_vec) {
  # One-hot encode
  lvs <- levels(factor_vec)
  one_hot_encoded <- data.frame(matrix(0, nrow = length(factor_vec), ncol = length(lvs)))
  colnames(one_hot_encoded) <- lvs
  for (i in 1:length(factor_vec)) {
    col_idx <- as.integer(factor_vec[i])
    one_hot_encoded[i, col_idx] <- 1
  }
  return(one_hot_encoded)
}
df <- read.csv("https://kaggle-getting-started.s3-ap-northeast-1.amazonaws.com/titanic/train.csv")

Key takeaways and discussion

Factor is built on top of integer

Source: Advanced R, Second Edition

In base R, many base functions use factor as the primary vector types instead of character

Choosing between character and factor

  • In preprocessing stage, use character for non-numerical attributes
  • In modeling stage, convert them into factor using factor() function

There are 2 important parameters in factor() function

  • levels allows user to customize encoding order
  • ordered allows user to create ordinal factor
